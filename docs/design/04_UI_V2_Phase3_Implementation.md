# Ace Career UI V2 · Phase 3 — Implementation Spec

Version: 1.0
Status: Ready for Codex
Priority: P0

---

## 一、阶段目标

重构「经营」中档案、团队、装备、赞助、媒体五个系统的页面结构，保持统一视觉语言，强化玩家身份/当前状态/关键决策，将低频内容下沉到二级页面。

**本阶段不要修改：** 教练能力计算/市场刷新/工资计算/装备加成公式/升级规则/赞助解锁/收入计算/媒体属性增长/精神属性计算/比赛模拟/周推进/数据结构/存档/Phase 1/Phase 2。

---

## 二、整体设计原则

- 五个模块统一 Design System，不同模块只改变信息层级和布局
- **禁止**不同模块使用不同主题色（蓝/红/金/紫）
- 每个模块只突出一个核心问题：
  - 档案：我是谁
  - 团队：谁在帮我
  - 装备：我在用什么
  - 赞助：我赚多少
  - 媒体：公众怎么看我

---

## 三、各模块页面结构

### 档案
```
球员身份 Hero → 当前职业状态 → 基础资料 → 职业概览 → 球风与能力摘要 → 查看完整生涯/球员能力
```

### 团队
```
当前团队 Hero → 4个核心职位(主教练/体能师/理疗师/经纪人) → 支持团队 → 团队效果 → 团队成本 → 教练市场入口
```
- 每个职位使用 ACRoleSlotCard（已签约/未签约双状态）
- 团队效果一张卡片统一展示，使用 ACAttributeBar（基础灰/有效绿/负面黄）
- 教练市场进入二级页面

### 装备
```
当前装备 Hero → 当前配置 → 属性影响 → 升级进度 → 装备库入口
```
- 升级显示 Lv.3→4 + 属性变化 + 费用
- 升级确认用 Sheet + 二次确认
- 装备库/比较用 ACComparisonSheet

### 赞助
```
商业收入 Hero → 当前合同（按类型分组）→ 待处理报价(最多3条) → 合作机会/赞助市场入口 → 合同历史入口
```
- 收入 Hero：每周固定/赛季累计/品牌数/商业等级
- 合同卡片：品牌/类型/周收入/奖金/排名要求/剩余时间
- 接受替换合同须明确说明影响

### 媒体
```
公众形象 Hero → 当前评价 → 媒体指标 → 精神属性摘要(4项) → 个性倾向摘要 → 待处理媒体事件 → 完整属性/个性入口
```
- 公众评价用系统生成标签（默默无闻/成长新星/巡回赛焦点/国际明星/超级巨星）
- 精神属性摘要：镇定/专注/决心/媒体应对
- 个性倾向先用摘要标签（职业/稳健/纪律性强/抗压一般）再进详情
- 媒体事件仅在存在时展示

## 五、统一规则

- **Bottom Sheet**：编辑/更换装备/升级确认/签约/续约/解约/接受赞助/拒绝赞助/属性说明/个性说明/媒体回应
- **危险操作**（解约/终止合同/替换装备/放弃赞助）→ 系统 Destructive 样式
- **空状态**：标题 + 解释 + 可执行入口（不只"暂无数据"）
- **金额格式**：$1,200 / $1,200/周 / $1.2M 统一
- **合同时间**：剩余 18 周 / 2026 赛季结束 / 已到期
- **属性格式**：12 / +3 / ↑1 统一不混用

## 七、验收标准（32 项）

1. 五个模块仍位于经营单行分类栏
2. 统一 Design System
3. 无连续大量同尺寸白卡片堆叠
4. 档案首屏突出玩家身份
5. 档案不承担完整生涯数据库
6. 团队明确展示四个核心职位
7. 未签约职位显示空槽位
8. 教练市场进二级页面
9. 团队加成和成本可清楚查看
10. 装备首屏突出当前装备
11. 装备属性区分加成与球员能力
12. 升级显示前后变化
13. 装备库进二级页面
14. 赞助首屏商业收入+当前合同
15. 待处理报价≤3条
16. 完整赞助市场进二级页面
17. 媒体首屏公众形象非完整属性列表
18. 精神/个性有摘要与详情
19. 媒体事件仅在存在时展示
20. 所有操作统一 Sheet
21. 危险操作二次确认
22. 所有模块有空状态
23. 切换保留滚动位置
24. 不重复加载业务数据
25. 旧存档正常读取
26. 数值结果完全一致
27. 小屏无溢出
28. Dynamic Type 可用
29. Dark Mode 正常
30. 编译通过无警告
31. Phase 1/2 不被破坏
32. 不新增未确认功能
