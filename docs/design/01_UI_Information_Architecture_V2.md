# Ace Career UI & Information Architecture V2

Version: 2.0
Status: Final
Priority: P0

---

## 一、设计理念（Design Philosophy）

Ace Career 是一款**职业网球生涯模拟游戏**，而不是传统的体育管理工具。

玩家扮演的是一名真实职业网球运动员，在 ATP 世界中成长，而不是经营一家俱乐部。

**整体设计原则：**

- 长得像 Apple（简洁、统一、克制）
- 玩起来像 Football Manager（世界模拟、长期经营、持续推进）
- 信息密度高，但阅读压力低
- 所有页面保持统一设计语言，不同系统只调整布局，不改变视觉风格

---

## 二、一级导航（固定，不再增加）

全游戏始终保持四个一级页面：

| Tab | 作用 | 核心问题 |
|-----|------|----------|
| **今日** Today | 玩家每周最主要的操作中心 | "这一周我应该做什么？" |
| **世界** World | 整个 ATP 世界 | — |
| **生涯** Career | 玩家职业生涯数据库 | — |
| **经营** Management | 管理自己的职业资源 | — |

---

### 今日（Today）

- 当前赛季
- 当前周数
- 下一站赛事
- 本周计划
- 身体状态
- 待处理事项
- 世界摘要
- Continue（继续下一周）

### 世界（World）

- 新闻
- 世界赛事
- ATP 排名
- Race 排名
- 球员动态
- 伤病 / 新秀 / 退役
- 世界纪录

**这里不属于玩家，而属于整个网球世界。**

### 生涯（Career）

- 生涯总览
- 排名
- 赛历
- 比赛历史
- 荣誉
- 训练记录
- 长期目标

**这里只记录"我"。**

### 经营（Management）

- 档案
- 团队
- 装备
- 赞助
- 媒体

设置移动到右上角，不再作为经营模块的一部分。

---

## 三、导航规则

整个 App 只允许三层结构：

```
Tab → 列表/总览 → 详情
```

所有编辑/升级/签约/筛选/选择统一使用 **Bottom Sheet**，不继续 Push 新页面。

---

## 四、Header 规范

**只有首页（今日）使用完整 Header：**

```
2026赛季  ·  第28周  ·  职业巡回赛  ·  世界排名
```

**其它页面统一使用简洁 Header：**

```
世界  |  生涯  |  经营
```

不再重复周数/世界排名/巡回赛 —— 每个页面增加约 100px 有效空间。

---

## 五、Section 规范

所有页面统一采用：

```
SECTION
标题
一句说明
↓
内容
```

例如：

```
CAREER
未来四周
报名由截止排名决定
```

整个 App 保持一致。

---

## 六、Card 规范

整个游戏只使用一套 Card：

- 白色背景
- 统一圆角
- 统一阴影
- 统一边框
- 相同间距
- 相同字体

不同页面只改变内容布局。**禁止为了不同模块使用不同颜色风格。**

---

## 七、视觉语言（统一）

全游戏保持：

- 黑色文字
- 白色卡片
- 浅灰背景
- 绿色 Accent

统一：Typography / Radius / Button / Icon / Divider / Animation

**不要因为商业/媒体/装备而改变整体配色。**

---

## 八、今日（Today）重构

页面顺序调整为：

```
Hero
↓
下一站赛事
↓
本周计划
↓
身体状态
↓
待处理事项
↓
世界摘要
↓
Continue
```

训练、恢复、报名、预测不再单独放一排快捷按钮，**全部融入对应 Card 内**：

- 点击"本周计划"进入训练
- 点击"下一站赛事"进入报名
- 点击"身体状态"进入恢复

首页更加沉浸，更像一款游戏而不是工具。

---

## 九、世界（World）重构

顶部分类最多保留三个：

```
动态  |  赛事  |  排名
```

球员/纪录/新秀/退役等内容进入各自分类，**不继续增加横向 Tab。**

---

## 十、生涯（Career）重构

取消现在大量横向 Segment。生涯首页改为纵向入口：

```
生涯总览
↓
赛历与比赛
↓
排名与积分
↓
训练记录
↓
荣誉与历史
```

进入二级页面后再根据需要少量使用 Segment（如 ATP / Race），而不是所有分类全部堆在顶部。

---

## 十一、经营（Management）重构

使用单行横向导航：

```
档案  |  团队  |  装备  |  赞助  |  媒体
```

设置移动到右上角。**避免两排按钮造成拥挤。**

---

## 十二、页面模板

所有系统遵循统一模板：

| 页面类型 | 结构 |
|----------|------|
| 列表页 | Header → Section → List |
| 详情页 | Hero → Summary → Modules |
| 管理页 | Hero → Current Status → Management Modules → History |

以后所有页面都从这些模板衍生。

---

## 十三、各系统重点

| 系统 | 突出内容 |
|------|----------|
| 团队 | 人物 / 能力 / 合同 / 工资 |
| 装备 | 当前装备 / 升级 / 品牌 / 属性 |
| 赞助 | 当前合同 / 收入 / 品牌 / 续约 |
| 媒体 | 声望 / 粉丝 / 精神属性 / 个性成长 |
| 生涯 | 长期成长 / 数据 / 荣誉 / 时间轴 |
| 世界 | ATP 世界变化 / 新闻 / 排名 / 球员动态 |

---

## 十四、后续开发顺序（固定）

| Phase | 内容 |
|-------|------|
| Phase 1 — 架构 | 今日首页 / Header / Card / Section / Button / Continue 体验 |
| Phase 2 — 导航 | 世界 / 生涯 / 经营 重新整理导航结构 |
| Phase 3 — 内容 | 重做团队/装备/赞助/媒体/档案，统一视觉层级 |
| Phase 4 — 游戏感 | 奖杯陈列室 / 生涯时间轴 / ATP 年终颁奖 / 世界纪录 / 球员关系 / 媒体采访 / 年度回顾 / 世界事件 |

---

## 最终目标

Ace Career 不追求功能最多，而是追求即使未来拥有数十个系统，整个产品依然像一款完整、统一、成熟的 AAA 职业生涯模拟游戏。

设计语言保持 Apple 的克制与一致性，信息架构借鉴 Football Manager 的深度，让玩家始终清楚自己当前的目标、职业发展和世界变化，而不是在复杂的功能中迷失。
