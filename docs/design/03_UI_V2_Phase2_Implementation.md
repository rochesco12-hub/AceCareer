# Ace Career UI V2 · Phase 2 — Implementation Spec

Version: 1.0
Status: Ready for Codex
Priority: P0

---

## 一、阶段目标

1. 底部 Tab「动态」→「世界」、「我的」→「经营」
2. 重构「世界」「生涯」「经营」信息架构
3. 移除普通页面重复的大型赛季 Header
4. 统一导航模式
5. 保留现有全部功能、数据和业务逻辑

**本阶段不要修改：** 世界模拟/排名计算/球员数据库/赛历/荣誉结算/教练装备赞助媒体数值/存档/周推进/Phase 1 今日首页。

---

## 二、最终一级导航

```
今日  |  世界  |  生涯  |  经营
```

SF Symbols：`calendar.day.timeline.left` / `globe.asia.australia` / `chart.line.uptrend.xyaxis` / `person.crop.circle`

- 选中 Accent Green / 未选中系统次级色
- 四个图标视觉重量接近 / 不混搭填充和线性风格
- 顺序不得再调整

---

## 三、全局导航规范

- 三层结构：Tab → 二级页面 → 详情页
- 筛选/排序/编辑/签约/解约/升级/选择/确认 → 统一 Sheet
- 二级页面使用 NavigationStack + 系统返回
- 一级页面 Header 统一 `ACPageHeader(style: .compact)`
- 不再重复显示赛季/周数/巡回赛/排名

---

## 四、「世界」重构

### 三分区（固定，不横向滚动）
```
动态  |  赛事  |  排名
```

```swift
enum WorldSection: String, CaseIterable {
    case feed = "动态", events = "赛事", rankings = "排名"
}
```

### 动态
- Major News Card（冠军/世界第一/重大伤病/退役/纪录）
- Compact Feed Row（普通比赛/排名小幅变化/报名退赛/新秀）
- 按周分组（第28周/第27周...）
- 重要动态 Hero（大满贯冠军/No.1易主/顶级退役）不强刷
- 筛选 Sheet：全部/重大新闻/赛事结果/排名变化/伤病与退赛/新秀与退役/只看与我相关

### 赛事
- 本周赛事 → 即将开始 → 近期结束 → 完整巡回赛日历（二级页）
- 赛事卡片：名称/城市国家/级别/场地/当前阶段/状态
- 筛选 Sheet：级别/场地/地区/状态

### 排名
- ATP / Race 两个 Segment
- 前三名 Podium（克制标签 #1/#2/#3）
- 排名列表（排名/变化/头像/姓名/国籍/积分）
- 玩家定位条（底部固定，NR 显示"尚未获得 ATP 排名"）

## 五、「生涯」重构

### 删除旧多胶囊导航 → 改为纵向总览

```
ACPageHeader
球员生涯 Hero（头像/姓名/国籍/排名/生涯最高/冠军数）
当前职业目标（目标名称/进度/条件）
当前排名与赛季表现（2×2 Grid：ATP/Race/战绩/奖金）
最近比赛（最多3场）
生涯模块入口 ×5（赛历与比赛 / 排名与积分 / 训练记录 / 荣誉与纪录 / 生涯统计）
职业世界概况（底部紧凑单行：活跃/青年/退役球员数）
```

### 五个模块入口使用 ACNavigationCard

### 二级页面
- 赛历与比赛 → ATP/Race/积分明细
- 排名与积分 → 训练/恢复/趋势
- 训练记录 → 按类别分组
- 荣誉与纪录 → 冠军/最佳成绩/排名纪录/里程碑
- 生涯统计 → 职业战绩/场地战绩/级别战绩/奖金/冠军...

## 六、「经营」重构

### 单行横向导航
```
档案  |  团队  |  装备  |  赞助  |  媒体
```

右上角齿轮 → 设置（NavigationStack Push）

```swift
struct ACScrollableCategoryBar<Item: Hashable>: View
// 单行/横向滚动/选中项自动居中/不换行/无系统滚动条
```

### 各模块 Phase 2 只调整结构

| 模块 | 结构 |
|------|------|
| 档案 | Hero→基础资料→职业概览→生涯数据摘要 |
| 团队 | Hero→核心成员(教练/体能/理疗/经纪人)→教练市场入口 |
| 装备 | Hero→属性加成→升级入口→装备库入口 |
| 赞助 | 收入 Hero→当前合同→可签约报价→历史合同入口 |
| 媒体 | 公众形象 Hero→媒体指标→精神属性摘要→个性摘要→媒体事件入口 |

## 十二、验收标准（24 项）

1. 底部 Tab「今日｜世界｜生涯｜经营」
2. 世界只有三个一级分区
3. 世界顶部 Segment 不横向滚动
4. 新闻/赛事/排名功能正常
5. 生涯无七八个横向胶囊
6. 生涯首页纵向总览+五入口
7. ATP/Race 整合到排名与积分
8. 比赛历史/赛历整合到赛历与比赛
9. 经营单行导航
10. 设置移到右上角
11. 经营导航不换行
12. 一级页面无重复赛季 Header
13. 继续使用 Phase 1 Design System
14. 业务数据不变化
15. 旧存档正常打开
16. 深链无失效入口
17. 各分页保留滚动位置
18. 小屏无横向溢出
19. Dynamic Type 可用
20. Dark Mode 正常
21. 排名/新闻列表不卡顿
22. 无重复世界模拟
23. 编译通过无警告
24. Phase 1 今日首页不被破坏

---

## 十三、实施顺序

1. 修改底部 Tab → 2. 世界根页面+三 Segment → 3. 接入动态 Feed → 4. 接入赛事列表 → 5. 接入排名 → 6. 重构生涯首页 → 7. 五模块入口 → 8. 旧生涯→新导航映射 → 9. 经营根页面 → 10. 单行分类栏 → 11. 各模块接入 → 12. 设置右上角 → 13. 更新导航目标 → 14. 空状态 → 15. 分类/滚动状态保存 → 16. 适配 → 17. 回归测试

Phase 2 不重做团队卡片/装备展示/赞助合同/媒体属性的具体视觉——这些属于 Phase 3。
