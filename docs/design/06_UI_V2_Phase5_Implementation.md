# Ace Career UI V2 · Phase 5 — Implementation Spec

Version: 1.0
Status: Ready for Codex
Priority: P0

---

## 一、阶段定位

Phase 5 是 Ace Career UI V2 的收尾阶段。不再重构主要页面，不新增大型玩法系统。

**只解决：** 新玩家是否知道怎么玩 / 是否始终知道下一步 / 设计语言是否统一 / 加载/空/错误/锁定状态是否完整 / 无障碍/性能/存档安全 / App 是否达到发布完成度

---

## 二、首次启动流程

```
品牌启动页(≤1s) → 游戏简介(最多3页+跳过) → 创建球员 → 选择起点 → 初始能力 → 世界说明 → 第一周引导
```

---

## 三、创建球员（5 步）

| 步骤 | 内容 |
|------|------|
| 1. 身份 | 姓名（校验）/ 国籍（搜索 Sheet）/ 出生年份 / 头像（姓名缩写即可）|
| 2. 身体 | 身高/体重/持拍手/反手（说明：影响初始能力与成长倾向）|
| 3. 球风 | 模板选择（底线进攻/稳定防守/全面/发球主导/红土/快速场地）展示优势/弱点/适合场地 |
| 4. 起点 | ITF 起步 + 初始资金/排名/团队/首赛 |
| 5. 确认 | 完整摘要 → 开始职业生涯 → 创建后立即写入存档 |

退出时不无提示丢失输入。

---

## 四、第一周引导

三步聚焦高亮（非全屏覆盖）：

1. **下一站赛事** → 高亮卡片 + "这里显示你即将参加或可以报名的赛事"
2. **本周计划** → 高亮卡片 + "每周安排训练、恢复或休息"
3. **继续下一周** → 高亮 Continue + "安排完成后推进时间"

首次比赛引导独立触发（查看对手→比赛准备→开始比赛→赛后结算）。

## 五、上下文教学系统

**触发原则：** 玩家第一次遇到某系统时提示，每个教学只展示一次。可重新查看 → 设置/帮助与规则。

**帮助中心分类：** 开始游戏 / 每周安排 / 比赛与赛事 / ATP排名与积分 / 训练与恢复 / 团队 / 装备 / 赞助 / 媒体 / 存档与数据

**规则来源必须与游戏真实规则一致。**

## 七、核心状态规范

### Loading
- < 300ms 本地操作不显示 Loading
- 全屏仅用于：首次加载存档/创建新职业/大型存档迁移/赛季世界生成/恢复损坏存档
- Skeleton：世界动态/排名/赛事/球员详情/赞助市场/工作人员市场

### Error
必须包含：发生了什么 + 是否影响存档 + 怎么处理。存档写入失败是高优先级——"本次操作尚未保存，请不要关闭游戏"。

### Locked
必须展示原因。✅ "进入世界前 500 后解锁" / "还需要 $2,400" ❌ "未解锁"

### Empty
标题 + 解释 + 可选行动。禁止整页"暂无数据"。

### Save
自动保存状态：已保存/正在保存/保存失败。不每次小操作后弹出"保存成功"。核心操作必须防重。

## 八、设置页面

存档与数据 / 通知（仅真实支持的类型）/ 外观（跟随系统/浅色/深色 + 减少动效）/ 比赛展示 / 帮助与规则 / 隐私政策 / 用户协议 / 关于（版本号+产品介绍）

重新开始职业 → 风险说明 + 二次确认 → Destructive 样式

---

## 九、全局一致性审计

- **边距**：左右 16pt / Section 24pt / 卡片 12pt
- **圆角**：仅保留 10/16/22
- **字体**：Display/Title/Headline/Body/Caption → ACTypography
- **颜色**：全部来自 ACColors（Accent/Success/Warning/Danger/Disabled/PrimaryText/SecondaryText/Card/Background/Divider）
- **图标**：统一 SF Symbols 重量+语义
- **按钮**：仅保留 Primary/Secondary/Text/Destructive/Icon
- **交互**：右箭头=可进入/省略号=更多/齿轮=设置/筛选=Filter Sheet/编辑=Edit Sheet

## 十、无障碍适配

### Dynamic Type（4 级 + Accessibility 1）
经营分类栏仍可用 / 排名列表不溢出 / 2×2 Grid→单列 / 不 minimumScaleFactor 缩正文

### VoiceOver
排名变化/胜负/状态进度/装备属性变化/锁定原因/比分/里程碑 → 合并语义标签

### Dark Mode
语义颜色 / 背景非纯黑 / 卡片层次清晰 / 绿色 Accent 对比度够

### Reduce Motion
取消大范围动画 / 排名直接淡变 / 里程碑简单淡入 / 页面切换遵循系统设置

### 小屏
无横向溢出 / 无固定宽度裁切 / Continue 不遮挡 / Tab Bar 不重叠

## 十一、性能优化

- UI 只读已生成数据，页面打开不触发世界模拟/排名结算/赛季生成
- 全部长列表使用 LazyVStack
- 赛季统计/排名趋势/团队加成/商业收入摘要 → ViewModel 预计算缓存
- SwiftUI body 内禁止：排序大数组/聚合多年数据/解析大存档/写入存档
- 图片：缩略图+异步+缓存+失败时快速占位

---

## 十二、数据一致性

世界排名/Race/赛事结果/职业战绩/冠军数/奖金/团队工资/装备等级/赞助收入/媒体属性/里程碑/赛季总结——同一数据在所有页面必须来自同一个数据源。

---

## 十三、真实球员数据保护

- 现实冠军不归零 / 历史最高排名不归零 / 大满贯纪录不归零
- 游戏新夺冠 ≠ "职业首冠"
- "赛季首冠"与"职业首冠"严格区分
- 玩家原创球员正常使用职业首次逻辑

---

## 十四、存档迁移

```
不完整存档 → 尝试迁移 → 自动备份 → 修复可确定字段 → 无法修复时提示用户
```

迁移失败 → "存档更新未完成，原始存档仍然保留"（不清空原档）

## 十八、Phase 5 验收标准（34 项）

核心：
1. 新玩家可无需外部说明完成创建+第一周
2. 首次引导不一次介绍全部功能
3. 每个教学只自动出现一次 + 帮助中心可重看
4. 创建球员不是单页长表单 + 退出不丢失内容
5. 所有页面统一 Design Token + 无散落圆角/颜色
6. 全部列表有 Loading/Empty/Error 状态
7. 锁定功能展示明确解锁条件
8. 存档保存失败不误报成功 + 关键操作不重复执行
9. 旧存档迁移前自动备份 + 失败不清空原档
10. Dynamic Type/VoiceOver/Dark Mode/Reduce Motion/小屏全部适配
11. 同一统计在所有页面一致
12. 真实球员历史与模拟历史严格区分
13. Debug 功能不在正式版可见
14. Phase 1-4 不被破坏
15. App 可达正式发布状态
